B단계별 빌드

01

데이터부터 눈으로 확인

hand_tracking2 안의 normalized_data CHOP 뷰어를 열고 손을 흔들어보세요. 21개 점 × x·y·z = 채널 63개가 출렁이면 준비 완료. 값은 이미 0~1로 정규화되어 나옵니다.

02

Select CHOPselect_tip — 63개 중 2개만

파라미터값
CHOPs../hand_tracking2/normalized_data
Channel Namesh1:index_finger_tip:x h1:index_finger_tip:y
왜? 매핑에 필요한 건 검지끝 x, y 딱 두 개. 나머지 61개는 소음입니다. h1 = 첫 번째로 잡힌 손(왼/오 보장 없음). z까지 쓰고 싶으면 패턴 h1:index_finger_tip:*.
03

Math CHOPmath_range — 쓰는 범위만 0~1로

파라미터 (Range 페이지)값
From Range0.15 ~ 0.85
To Range0 ~ 1
왜? 손은 화면 가장자리까지 못 갑니다 — 실제로는 0.15~0.85만 씀. 그 구간을 0~1로 늘려야 커서가 화면 전체를 커버합니다. 이 두 값이 감도 노브.
거울 반전은? MediaPipe COMP의 Horizontal Flip이 이미 처리 중이라 여기선 안 건드립니다.
04

Rename CHOPrename_hand — 채널명에 의미 부여

파라미터값
Fromh1:index_finger_tip:x h1:index_finger_tip:y
Tohandx handy
왜? 이후 익스프레션이 op('null_hand')['handx']처럼 읽히게 됩니다. 네트워크가 커질수록 이 습관이 생명.
05

Lag CHOPlag_smooth → Null CHOPnull_hand

Lag 0.12 / 0.12. 가만히 있어도 파르르 떨리는 지터를 다림질합니다. 마지막에 Null을 붙이면 정규화 척추 완성 — 매핑은 전부 이 null_hand에서 뽑아 씁니다.

Lag vs Filter? Lag = 목표를 따라가는 관성(즉각 반응, 부드러운 꼬리) · Filter = 최근 값의 평균(더 매끈, 전체가 살짝 늦음). 커서엔 Lag. 둘 다 걸어보고 눈으로 고르세요.
06

Circle TOPcircle_dot — 허공 커서

파라미터값
Resolution1280 × 720 (Common 페이지, Output Resolution = Custom)
Radius X0.02  (Radius Y에는 me.par.radiusx — 항상 정원)
Center Xop('null_hand')['handx'] - 0.5
Center Yop('null_hand')['handy'] * -1 + 0.5  (= 0.5 - handy)
Softness0.005
좌표계 번역. MediaPipe는 왼쪽 위가 (0,0), 아래로 갈수록 y 증가. TD의 Circle은 화면 중앙이 (0,0), 위로 갈수록 +. 그래서 x는 -0.5(중심 이동), y는 0.5 - handy(뒤집기 + 중심 이동).
MediaPipe            TD Circle
(0,0)┌──────┐        ┌──────┐
     │      │   →    │ (0,0)│   x' = x - 0.5
     │      │        │  +   │   y' = 0.5 - y
     └──────┘(1,1)   └──────┘
07

Feedback + Level + Add잔상 루프 — 지난주 그 구조

  • circle_dot → add_trail의 1번 입력
  • feedback1 → level_fade → add_trail의 2번 입력
  • feedback1의 Target TOP = add_trail (루프를 닫는 열쇠)
  • level_fade의 Opacity = 0.93 → 잔상 길이 노브
  • add_trail → null_out, 뷰어 켜기 — 완성!
원리. Feedback은 직전 프레임의 add_trail을 기억했다가 되돌려줍니다. 매 프레임 93%로 어두워지며 쌓이니까 → 움직인 궤적이 꼬리처럼 남습니다. 1.0이면 화면이 영영 안 지워져요.
