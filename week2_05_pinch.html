05_Pinch

연속값(거리) → 스위치(0/1) → 봉투(부드러운 산) — 3단 변환. 신디사이저의 엔벨로프와 같은 물건입니다.

B핵심 개념: 연속값을 이벤트로

지금까지의 매핑은 전부 연속이었습니다 (손 높이 = 음정처럼 붙어서 따라가기). 그런데 "집는 순간 뭔가 일어나기"는 다른 종류의 문제 — 이벤트가 필요합니다:

핀치 거리     0.21  0.18  0.12  0.04  0.03  0.05  0.15   연속값
Logic         0     0     0     1     1     1     0      스위치 (0.05 미만 = ON)
Trigger       0     0     0     1.0   0.7   0.4   0.2    봉투 (확 켜졌다 서서히)
왜 봉투? 스위치 0/1을 바로 비주얼에 꽂으면 껌뻑거리기만 합니다. Trigger가 스위치를 모양이 있는 곡선(어택-디케이)으로 바꿔줘야 "펀치"의 맛이 생깁니다. 드럼 소리가 '딱' 하고 사라지는 그 모양.

02

Logic CHOPlogic_pinch — ★새 개념: 스위치

파라미터값
Convert InputOn When Value is in Bounds
Bound Min / Max0 / 0.05
Logic = 관문. "거리가 0~0.05 사이인가?" → 예: 1, 아니오: 0. 아날로그 세계(거리)가 디지털 세계(집었다/안집었다)로 건너오는 다리입니다.
03

Trigger CHOPtrigger_flash — ★새 개념: 봉투

파라미터값역할
Attack0.02확 켜지기 (펀치)
Decay0.4사그라드는 시간
Sustain0누르고 있어도 유지 안 함
Release0.3떼었을 때 마무리

이어서 Rename *→flash, null_pinch로 마무리.

파라미터값

집는 순간 원이 확 커지며 가장자리가 풀어졌다가, 0.4초에 걸쳐 사그라듭니다. Softness에도 flash를 꽂아서 커질수록 부드러워지는 디테일.

D안 될 때

  • 반응이 없어요 — 1단계로 돌아가 실측. 손 크기·카메라 거리에 따라 "집었을 때 값"이 다릅니다 (0.02~0.08 사이). Bound Max를 거기에 맞추세요.
  • 연발로 터져요 — 경계값 근처에서 스위치가 떨리는 것. Logic 앞에 Lag(0.05)를 넣거나 Trigger의 Re-Trigger Delay를 0.3으로.
  • 손을 폈는데도 켜져 있어요 — Sustain이 0인지 확인.
  • 트래킹이 자꾸 끊겨요 — 조명을 밝게, 배경과 손의 대비 확보, 카메라에서 50cm~1m. 품질의 절반은 조명입니다.
  • 값이 아예 안 움직여요 — hand_tracking2/helpers의 h1:hand_active가 1인지 먼저 확인. 0이면 손 인식 자체가 안 되는 것.
  • Select에 노란 경고 — CHOPs 경로는 노드의 부모 네트워크 기준입니다. 베이스 안에서는 ../hand_tracking2/… (../../ 아님!)

E더 가보기 — 제스처는 이미 준비되어 있다

hand_tracking2의 gestures CHOP에는 학습된 제스처 채널이 통째로 들어 있습니다:

h1:Closed_Fist   h1:Open_Palm   h1:Pointing_Up
h1:Thumb_Up      h1:Thumb_Down  h1:Victory      h1:ILoveYou
  • 각 채널은 해당 제스처일 확률(0~1) — 같은 Logic → Trigger 패턴을 그대로 꽂으면 "브이 하면 터진다"가 됩니다.
  • 씬 전환기 — Open_Palm = 씬1, Closed_Fist = 씬2를 Switch TOP의 Index에.
  • 집어서 끌기 — h1:pinch_midpoint:x·y(집은 위치!)와 flash를 조합하면 물건을 집어 옮기는 인터랙션.
  • 사운드 — flash를 02_Theremin 오실레이터의 Amplitude에 곱하면 집을 때마다 "띵".
