B단계별 빌드

01

01_Hand의 척추 다섯 개를 복붙

select_tip → math_range → rename_hand → lag_smooth → null_hand를 그대로 복사해오세요 (드래그 선택 → Ctrl/Cmd+C·V). 새 개념 0개 — 그게 포인트입니다.

딱 하나 조정. lag_smooth를 0.15로 살짝 올리기 — 음정이 미끄러지듯 이어지는 글라이드(포르타멘토) 맛이 생깁니다. 테레민의 그 느낌.
02

Select CHOPselect_active — 손이 있을 때만

파라미터값
CHOPs../hand_tracking2/helpers
Channel Namesh1:hand_active

이어서 Lag lag_gate (0.05 / 0.3) → Null null_gate.

왜 게이트? hand_active는 손이 잡히면 1, 없으면 0. 이걸 볼륨에 곱하면 손이 없을 때 자동 무음. Lag을 비대칭(켜질 땐 0.05로 빠르게, 꺼질 땐 0.3으로 천천히)으로 걸면 뚝 끊기지 않고 페이드아웃됩니다.
03

Audio Oscillator CHOPosc_voice — 소리 그 자체

파라미터값
WaveformSine (일단)
Base Frequency220 * 2 ** ((1 - op('null_hand')['handy']) * 2)
Amplitudeop('null_gate')['h1:hand_active'] * op('null_hand')['handx'] * 0.4
왜 2의 거듭제곱? 귀는 로그로 듣습니다 — 한 옥타브 = 주파수 ×2. 손 높이를 지수에 넣어야 위아래 움직임이 균등한 음정 변화로 들립니다.
handy(손 높이)   1.0 ── 바닥  → 220 Hz  (A3, 낮은 라)
                0.5 ── 중간  → 440 Hz  (A4) ← ×2
                0.0 ── 꼭대기 → 880 Hz  (A5) ← ×2
볼륨 = hand_active(0/1) × handx(왼쪽 0 … 오른쪽 1) × 0.4
04

Audio Device Out CHOPaudio_out — 스피커로

osc_voice를 와이어로 연결, Volume 0.5. 손을 들어 올려보세요 — 음이 올라갑니다. 오른쪽으로 밀면 커지고, 손을 내리면(화면에서 빼면) 소리가 스르르 사라집니다.

C조절 노브

바꾸고 싶은 것어디를어떻게
기준음frequency의 220110 = 한 옥타브 아래 · 440 = 위
음역 폭frequency의 * 2* 3 = 세 옥타브 (더 드라마틱)
소리 성격WaveformSine = 순한 휘파람 · Saw = 브라스 · Square = 게임기
글라이드lag_smooth0.05 = 또렷 · 0.4 = 끈적한 포르타멘토

D안 될 때

  • 소리가 아예 안 나요 — ① 손이 화면에 있나요? (amp는 hand_active에 곱해져 있음) ② 손이 왼쪽 끝이면 handx=0이라 무음 ③ audio_out의 Device가 맞는 출력장치인지.
  • 음정이 부들부들 떨려요 — lag_smooth를 올리세요. 지터가 그대로 음정 비브라토가 되는 중.
  • 연습실이 시끄러워요 — 이어폰… 혹은 Volume 0.2. 서로의 테레민이 싸우면 아수라장 ㅎㅎ

E더 가보기

  • 음계 스냅 — 연속 음정 대신 반음 단위로: 220 * 2 ** (round((1 - op('null_hand')['handy']) * 24) / 12). round()가 반음(1/12 옥타브) 격자에 스냅시켜 줍니다.
  • 비브라토 — 다른 LFO CHOP(5~7Hz, 작은 진폭)를 frequency에 더하기.
  • 두 손 연주 — h2 체인을 만들어 두 번째 오실레이터 = 화음.
  • 빛과 함께 — 01_Hand와 동시에 켜면 연주가 그림이 됩니다. 미니 공유각.
